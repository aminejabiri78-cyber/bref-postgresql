# Superstore Sales Data Engineering Project

## Project Description

This project demonstrates the design and implementation of a **relational database system** for analyzing retail sales data using the **Superstore dataset**.

The goal is to simulate a **real-world Data Engineering workflow** that includes:

* Database design (normalized schema)
* Data cleaning and preprocessing
* Data loading into PostgreSQL
* SQL transformations
* Creation of analytical views for business insights

This project highlights essential **data engineering and analytics skills** using Python and SQL.

# Database Design

The database follows a **normalized relational model** to reduce redundancy and improve data integrity.

## Tables

### Customer

Stores customer information.

| Column        | Description                |
| ------------- | -------------------------- |
| Customer_ID   | Unique customer identifier |
| Customer_Name | Customer name              |
| Segment       | Customer segment           |
| Cat_Client    | Customer category          |

---

### Location

Stores geographical information.

| Column      | Description  |
| ----------- | ------------ |
| Postal_Code | Postal code  |
| Country     | Country      |
| City        | City         |
| State       | State        |
| Region      | Sales region |

---

### Orders

Stores order information.

| Column      | Description             |
| ----------- | ----------------------- |
| Order_ID    | Unique order identifier |
| Order_Date  | Date of order           |
| Ship_Date   | Shipping date           |
| Ship_Mode   | Shipping method         |
| Customer_ID | Customer reference      |
| Postal_Code | Shipping location       |

---

### Product

Stores product information.

| Column       | Description          |
| ------------ | -------------------- |
| Product_ID   | Product identifier   |
| Product_Name | Product name         |
| Category     | Product category     |
| Sub_Category | Product sub-category |

---

### Order Details

Stores transaction details.

| Column     | Description       |
| ---------- | ----------------- |
| Order_ID   | Order reference   |
| Product_ID | Product reference |
| Quantite   | Quantity          |
| Sales      | Sales amount      |
| Cost       | Product cost      |

Primary Key: **(Order_ID, Product_ID)**

# Analytical SQL Views

To support business analysis, several **SQL views** were created.

### Sales per Category

Performance comparison between product categories.

### Sales per Region

Sales distribution across different regions.

### Average Profit per Order

Measures the profitability of orders.

### Average Profit per Customer

Analyzes customer profitability.
